ทฤษฎีพื้นฐาน (Fundamentals)

รวมทฤษฎีที่จำเป็นสำหรับ Network Engineer Intern ก่อนเริ่มทำ Lab หรือก่อนสัมภาษณ์

1. OSI Model & TCP/IP Model

การเข้าใจ Layer เป็นหัวใจของการทำ Troubleshooting

  • Layer 1 (Physical): สายแลน, พอร์ต (up/down), สัญญาณไฟฟ้า (Check: show ip int brief)
  • Layer 2 (Data Link): MAC Address, Switch, VLAN, ARP (Check: show mac address-table, show vlan)
  • Layer 3 (Network): IP Address, Router, Routing Table (Check: show ip route)
  • Layer 4 (Transport): TCP (เชื่อถือได้), UDP (รวดเร็ว), Port Numbers
  • Layer 7 (Application): HTTP, DNS, DHCP

2. L2 Switching & MAC Address Table

Switch ทำงานที่ Layer 2 ตัดสินใจส่งข้อมูลโดยดูจาก Destination MAC Address

  • Learning: Switch เรียนรู้ Source MAC Address จาก Frame ที่เข้ามา และบันทึกลง MAC Table คู่กับ Port นั้น
  • Forwarding: ถ้ารู้จัก Destination MAC ก็จะส่งออกไปเฉพาะ Port ปลายทาง
  • Flooding: ถ้าไม่รู้จัก Destination MAC (Unknown Unicast) หรือเป็น Broadcast/Multicast จะส่งออกทุก Port ยกเว้น Port ที่รับมา

3. VLAN & Trunking

VLAN (Virtual LAN) คือการแบ่ง Broadcast Domain ออกเป็นส่วนๆ ภายใน Switch ตัวเดียวกัน

  • Access Port: พอร์ตที่อยู่ใน VLAN เดียว มักต่อกับ PC (Untagged traffic)
  • Trunk Port: พอร์ตที่ส่งข้อมูลได้หลาย VLAN มักใช้เชื่อมระหว่าง Switch หรือ Router (Tagged traffic ด้วย 802.1Q)
  • Native VLAN: VLAN ที่ส่งผ่าน Trunk Port โดยไม่แปะ Tag (ค่าเริ่มต้นคือ VLAN 1) ต้องตรงกันทั้งสองฝั่ง

4. ARP (Address Resolution Protocol)

โปรโตคอลที่ใช้หา MAC Address เมื่อรู้ IP Address ปลายทาง (ถ้าอยู่ใน Subnet เดียวกัน) หรือหา MAC ของ Gateway (ถ้าอยู่คนละ Subnet)

  • ARP Request: ส่งเป็น Broadcast (FF:FF:FF:FF:FF:FF) "ใครคือ IP นี้ ช่วยบอก MAC Address มาหน่อย"
  • ARP Reply: ส่งกลับเป็น Unicast "ฉันเอง นี่คือ MAC Address ของฉัน"

5. Routing Table & Longest Prefix Match

Router ทำงานที่ Layer 3 ตัดสินใจส่งข้อมูลโดยดูจาก Destination IP Address เทียบกับ Routing Table

  • Longest Prefix Match (LPM): กฎข้อแรกของการ Routing! Router จะเลือกเส้นทางที่มี Subnet Mask ยาวที่สุด (เจาะจงที่สุด) เสมอ เช่น ปลายทาง 10.1.1.5 จะเลือก /24 ก่อน /16
  • Administrative Distance (AD): ความน่าเชื่อถือของ Source (Connected=0, Static=1, OSPF=110) ใช้ตัดสินเมื่อได้เส้นทางจากหลาย Protocol
  • Metric: ระยะทางหรือต้นทุน ใช้ตัดสินเมื่อได้เส้นทางจาก Protocol เดียวกันแต่มีหลายทาง

6. วิธีคิด Troubleshooting สไตล์ Engineer

เวลาเจอปัญหา "Ping ไม่ได้" ห้ามเดามั่ว ให้ไล่ตรวจทีละ Layer ตามแนวทาง Divide and Conquer หรือ Bottom-Up

  1. เช็ค Layer 1: สายเสียบแน่นไหม? พอร์ตสถานะ up/up หรือเปล่า?
  2. เช็ค Layer 2: อยู่ถูก VLAN ไหม? Trunk อนุญาต VLAN นี้ไหม? ทราบ MAC Address (ARP) ไหม?
  3. เช็ค Layer 3 ขาไป: เครื่องต้นทางตั้ง Gateway ถูกไหม? Router มี Route ไปปลายทางไหม?
  4. เช็ค Layer 3 ขากลับ: เครื่องปลายทางมี Gateway หรือไม่? Router ฝั่งปลายทางมี Return Route กลับมาหาเครื่องต้นทางหรือไม่? (สำคัญมาก คนมักลืมขากลับ)
