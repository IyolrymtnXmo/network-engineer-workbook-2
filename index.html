🧪 Labs ทั้งหมด

LAB 01 Static Routing พื้นฐาน เชื่อมสองสาขาด้วย static route — เข้าใจ routing table, next-hop และเหตุผลที่ต้องมี route ครบทั้งสองทิศ พื้นฐาน ⏱ 60 นาที LAB 02 Static + Default Route และ Longest Prefix Match สามไซต์ + ISP จำลอง — default route, summary route และกติกาการเลือกเส้นทางของ router พื้นฐาน–กลาง ⏱ 75 นาที LAB 03 VLAN และ Trunking แยกทราฟฟิก HR / IT ด้วย VLAN ข้ามสอง switch — access port, trunk 802.1Q และ native VLAN พื้นฐาน ⏱ 60 นาที LAB 04 Router-on-a-Stick ให้ VLAN คุยกันด้วย router พอร์ตเดียว — subinterface, encapsulation dot1Q และเส้นทาง frame แบบ hairpin กลาง ⏱ 75 นาที LAB 05 Inter-VLAN Routing ด้วย Layer 3 Switch (SVI) อัพเกรดจาก ROAS เป็น L3 switch — ip routing, SVI, routed port และการเลือกดีไซน์ในงานจริง กลาง ⏱ 90 นาที LAB 06 ARP และ MAC Address Table เจาะลึก เห็น ARP กับตาทุกจังหวะ — ARP table vs MAC table, duplicate IP, mask ผิดแล้ว ARP เพี้ยนยังไง กลาง ⏱ 75 นาที LAB 07 Layer 2 Troubleshooting เครือข่ายพังมาให้แล้ว 5 จุด — ไล่หาจาก ticket ผู้ใช้จริงด้วยลำดับการตรวจ L2 อย่างเป็นระบบ กลาง–สูง ⏱ 90 นาที LAB 08 Layer 3 Troubleshooting route ผิด next-hop, ขากลับหาย, mask ทับซ้อน — ฝึกอ่าน traceroute และไล่ routing สองทิศทาง กลาง–สูง ⏱ 90 นาที LAB 09 Mixed Troubleshooting (L1–L3) งาน change กลางคืนพัง 7 จุดคละทุกชั้น — divide & conquer, follow-the-path และการปิดงานด้วย verification matrix สูง ⏱ 120 นาที LAB 10 Final Assessment — ออกแบบ ติดตั้ง และกู้ระบบ ข้อสอบรวบยอดจับเวลา: ออกแบบ VLSM จาก requirement จริง, config ทั้งเครือข่าย, กู้ 2 ticket และจำลองสัมภาษณ์ 10 ข้อ ข้อสอบรวบยอด ⏱ 150 นาที

📚 เอกสารประกอบ

🧠 ทฤษฎีพื้นฐาน OSI, L2 switching, VLAN, ARP, routing table, Longest Prefix Match, Packet Flow ฉบับสมบูรณ์ และวิธีคิด troubleshooting — อ่านทวนก่อนเจอ reviewer ⌨️ Command Reference คู่มือคำสั่ง Cisco IOS ทุกคำสั่งที่ใช้ใน workbook — syntax, ตัวอย่าง output, วิธีอ่าน และ red flags ที่ต้องสังเกต

🛠 เครื่องมือที่แนะนำ

Workbook นี้ออกแบบให้ทำตามได้ในเครื่องมือจำลองเครือข่ายทั่วไป เลือกอย่างใดอย่างหนึ่ง:

  • Cisco Packet Tracer — ฟรีผ่าน Cisco NetAcad เหมาะที่สุดสำหรับผู้เริ่มต้น ทุก Lab ทำได้ครบ
  • GNS3 / EVE-NG + IOSv — เหมือนอุปกรณ์จริงที่สุด เหมาะกับ Lab 7–10 ที่เน้น troubleshooting
  • อุปกรณ์จริง — ถ้าที่ออฟฟิศมี switch/router ให้ยืม ยิ่งดี (ขออนุญาตพี่ในทีมก่อนเสมอ)
💡
เคล็ดลับจากพี่ Reviewer สิ่งที่ทำให้ intern โดดเด่นไม่ใช่การ config เร็ว แต่คือการอธิบายได้ว่า "packet เดินทางยังไง และทำไมถึง config แบบนี้" — ทุกครั้งที่ทำ Lab เสร็จ ลองเล่า packet flow ให้ตัวเองฟังหนึ่งรอบ
